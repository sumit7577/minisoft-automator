Registers a Windows Hello / passkey key against the device registration service. Needs an access token scoped to urn:ms-drs:enterpriseregistration.windows.net — if the token below is left blank, roadtx falls back to whatever's cached in .roadtools_auth, which will fail with a "wrong token audience" error unless that token was requested for this specific resource.

          
          
          
          
        


        

          
Join, register, or delete a device object using an access token scoped to the device registration service. Needs a token minted via Refresh → Resource first (or paste one below), same prerequisite as WinHello. join/register write a device cert + key locally that the PRT / PRT-auth flows then consume.

          

            ⚠️
            “join” and “register” create a real device object in the target tenant. This is a live, attributable change to the environment — only run it against a tenant you’re authorized to modify. “delete” removes one using the cert/key below.
